# Why write tests?

**Target time:** 30 seconds

---

## Talk track

> Tests give **confidence to change code** — refactor, upgrade deps, ship fixes without fear.
>
> They **document behavior** — next dev reads test to see expected outcome.
>
> They **catch regressions** before prod — especially logic bugs and edge cases humans retest poorly.
>
> I don't test for 100% coverage religion — I test **high-value paths**: money flows, auth, data transforms, bugs that burned us once.
>
> ROI: fewer hotfixes, faster code review ("green CI" baseline).

---

## Avoid

- "Tests guarantee no bugs" — they reduce risk
- "We don't need tests, we have QA" — different layers
