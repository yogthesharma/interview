# Flask vs Django — tradeoffs for a small team?

**Target time:** 45 seconds

---

## Talk track

> **Django:** admin, ORM, auth, migrations built-in — faster CRUD-heavy internal apps, more conventions.
>
> **Flask:** minimal core — you choose SQLAlchemy, auth, structure — more flexibility, more decisions.
>
> **Small API team:** Flask or FastAPI + SQLAlchemy is lean. **Small team + admin UI day one:** Django wins.
>
> **Hireable take:** pick based on product shape, not religion.

---

## Avoid

- "Django is always too heavy" — wrong for many products
