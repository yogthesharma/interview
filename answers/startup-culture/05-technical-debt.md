# How do you handle technical debt?

**Target time:** 30 seconds

---

## Talk track

> **Acknowledge it openly** — debt isn't shame, it's a tradeoff we took for speed.
>
> **Tie paydown to pain** — fix debt when it's **slowing the team** or causing incidents, not aesthetic refactors.
>
> **Boy scout rule** — leave touched files a bit better; don't boil the ocean in one PR.
>
> **Document** — ticket for bigger items ("migrate X to React Query") so it's planned, not forgotten.
>
> Example: at IQM we carried legacy class components until the dashboard bug forced a **focused migration** — debt paid when risk justified it.

---

## Avoid

- "We should rewrite everything"
- Never paying debt down
