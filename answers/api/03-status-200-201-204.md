# When do you return 200 vs 201 vs 204?

**Target time:** 30 seconds

---

## Talk track

> - **200 OK** — success with **body** — GET, PATCH, PUT, POST when returning resource is optional  
> - **201 Created** — **POST** created a new resource — include `Location` header + body with new id  
> - **204 No Content** — success, **no body** — DELETE success, PUT/PATCH when you don't return entity
>
> **Rule of thumb:** created something new → **201**. Success but nothing to return → **204**. Default success with JSON → **200**.

---

## Code

```ts
// POST create
reply.code(201).header("Location", `/v1/applications/${app.id}`).send(app);

// DELETE
reply.code(204).send();

// GET / PATCH
reply.code(200).send(application);
```

---

## Avoid

- 200 on POST create when REST clients expect 201
