# How do you do code review in your team?

**Target time:** 30 seconds (see also `background/18-code-reviews.md`)

---

## Talk track

> **Process:** PR with description + screenshots for UI, link ticket, tag reviewers. Usually **1 approval** required; bigger changes got a second pair of eyes.
>
> **Culture:** Async on GitHub — comments, suggestions, resolve threads. Blockers vs nits labeled. I wasn't formal CODEOWNERS lead but people often pulled me in on **React, TS, API** PRs after our migration.
>
> **What I look for:** correctness, edge cases, consistency with squad patterns, perf on hot paths. I ask questions more than dictate.
>
> **Receiving:** not personal, push back with reasoning, thank people for catching bugs pre-prod.

---

## Avoid

- Rubber-stamping
- "Only lead reviews" if you participated actively
