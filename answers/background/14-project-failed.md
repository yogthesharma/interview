# Describe a project that failed or didn't go as planned.

**Target time:** 60–90 seconds (STAR — show humility + learning)

---

## Talk track (TypeScript migration — phased plan took longer than hoped)

> **Situation:** The **TypeScript migration at IQM** — I was a big advocate and helped drive it. Good outcome eventually, but **the plan didn't go as smoothly as I sold it**.
>
> **What went wrong:** I underestimated how long **shared legacy utilities** would take — untyped helpers used everywhere, weird dynamic patterns. My initial timeline assumed modular migration; reality was **whack-a-mole** of type errors across dependent packages.
>
> **Impact:** We slipped on a couple of feature deadlines because eng bandwidth went to migration fixes. Some teammates frustrated — felt like "Yog's project" slowing them down.
>
> **What I did:**
> - Reset expectations with the squad — honest timeline, not optimism
> - Prioritized **API boundaries and high-churn modules** first instead of 100% coverage day one
> - Paired with others so it wasn't solo hero work
> - Documented patterns so the next module was faster
>
> **Learning:** Big technical bets need **incremental wins visible early**. Influence as an IC means bringing people along, not just being technically right.

## Alternative (lighter)

**Early perf refactor at Atlys** — optimized one route but introduced a regression on another step; caught in QA; rolled back; added regression checklist. Shows you ship mistakes and fix process.

- Project that truly failed with no learning
- Blaming the team
